# 配置文件管理完整指南

## 📚 功能总览

v2.8.0 提供完整的配置文件管理系统，包括：

1. **💾 保存配置** - 保存当前规则为配置文件
2. **📂 加载配置** - 从历史配置快速应用
3. **🗑️ 删除配置** - 删除不需要的配置
4. **📤 导出配置** - 导出配置文件到任意位置
5. **📥 导入配置** - 从文件导入配置
6. **📚 模板库** - 使用预置模板快速开始
7. **🔗 生成分享** - 生成配置分享给团队

## 🎯 使用场景

### 场景 1：多环境管理

**问题：** 需要频繁切换开发、测试、生产环境

**解决方案：**
```
1. 配置开发环境变量
2. 保存为"开发环境"
3. 配置测试环境变量
4. 保存为"测试环境"
5. 配置生产环境变量
6. 保存为"生产环境"

使用时：
- 下拉选择"生产环境"
- 一键应用所有规则
- 点击"开始替换"
```

### 场景 2：团队协作

**问题：** 团队成员需要使用相同的配置

**解决方案：**
```
开发者 A：
1. 配置好生产环境规则
2. 点击"🔗 生成分享"
3. 复制配置或保存为文件
4. 发送给团队成员

开发者 B：
1. 收到配置文件
2. 点击"📥 导入"
3. 选择收到的配置文件
4. 配置自动加载，可以直接使用
```

### 场景 3：多渠道包

**问题：** 需要为不同渠道生成不同的包

**解决方案：**
```
保存多个配置：
- "Google Play" - 图标、变量、渠道ID
- "App Store" - 图标、变量、渠道ID
- "华为商店" - 图标、变量、渠道ID

打包时：
1. 选择对应渠道配置
2. 所有规则自动填充
3. 一键生成渠道包
```

## 💾 保存配置

### 步骤

1. **配置规则**
   - 添加类和变量
   - 或配置图片映射

2. **点击保存**
   - 点击"💾 保存"按钮
   
3. **输入名称**
   - 输入配置名称（如"生产环境"）
   
4. **完成**
   - 配置已保存
   - 可在下拉框看到

### 示例

```
配置名称: 生产环境

包含内容:
- AppConfig 类
  - API_URL = "https://prod-api.com"
  - DEBUG_MODE = false
- SecurityConfig 类
  - API_KEY = "prod_key_12345"

保存后显示:
生产环境 (2026-02-05 14:30)
```

## 📂 加载配置

### 步骤

1. **打开下拉框**
   - 点击配置下拉框
   
2. **选择配置**

   - 选择需要的配置
   
3. **自动填充**
   - 规则自动填充到界面
   - 可以直接使用或修改

## 🗑️ 删除配置

### 步骤

1. **选择配置**
   - 在下拉框选择要删除的配置
   
2. **点击删除**
   - 点击"🗑️ 删除"按钮
   
3. **确认**
   - 确认删除操作
   
4. **完成**
   - 配置文件已删除
   - 从列表中移除

## 📤 导出配置

### 用途
- 备份配置
- 分享给团队
- 保存到其他位置

### 步骤

1. **选择配置**
   - 在下拉框选择要导出的配置
   
2. **点击导出**
   - 点击"📤 导出"按钮
   
3. **选择位置**
   - 选择保存位置
   - 输入文件名
   
4. **完成**
   - 配置已导出为 JSON 文件

### 导出文件示例

**文件名：** `生产环境.json`

**内容：**
```json
{
  "platform": "ios_variable",
  "name": "生产环境",
  "createdAt": "2026-02-05T14:30:00.000Z",
  "data": [
    {
      "className": "AppConfig",
      "expanded": true,
      "variables": [
        {
          "name": "API_URL",
          "value": "\"https://prod-api.com\""
        },
        {
          "name": "DEBUG_MODE",
          "value": "false"
        }
      ]
    }
  ]
}
```

## 📥 导入配置

### 用途
- 导入团队成员分享的配置
- 恢复备份的配置
- 从其他电脑迁移配置

### 步骤

1. **点击导入**
   - 点击"📥 导入"按钮
   
2. **选择文件**
   - 选择配置 JSON 文件
   
3. **自动导入**
   - 配置自动导入并加载
   - 可在下拉框看到
   
4. **使用配置**
   - 规则已自动填充
   - 可以直接使用

### 支持的文件

- `.json` 格式
- 必须是本工具导出的配置文件
- 支持所有平台和类型的配置

## 📚 模板库

### 用途
- 快速开始配置
- 学习配置示例
- 基于模板修改

### 内置模板

#### iOS 变量模板

**1. 生产环境配置模板**
```
AppConfig:
- API_URL = "https://api.production.com"
- API_TIMEOUT = 30
- DEBUG_MODE = false
- LOG_LEVEL = "ERROR"

SecurityConfig:
- ENABLE_SSL = true
- API_KEY = "YOUR_PRODUCTION_API_KEY"
```

**2. 测试环境配置模板**
```
AppConfig:
- API_URL = "https://api.test.com"
- DEBUG_MODE = true
- LOG_LEVEL = "DEBUG"
```

#### Android 变量模板

**1. 生产环境配置模板**
```
Constants:
- BASE_URL = "https://api.production.com"
- DEBUG = false
- VERSION_NAME = "1.0.0"
```

**2. 多渠道配置模板**
```
ChannelConfig:
- CHANNEL_ID = "google_play"
- CHANNEL_NAME = "Google Play"
- UPDATE_URL = "https://play.google.com/..."
```

#### 图片模板

**iOS/Android App 图标替换模板**
```
AppIcon.png → new_app_icon.png
LaunchImage.png → new_launch_image.png
```

### 使用步骤

1. **打开模板库**
   - 点击"📚 模板库"按钮
   
2. **浏览模板**
   - 查看所有可用模板
   - 阅读模板说明
   
3. **选择模板**
   - 点击"✅ 使用此模板"
   
4. **自动应用**
   - 模板规则自动填充
   - 可以继续修改

## 📁 文件存储位置

### macOS
```
~/Library/Application Support/ios-android-refactor-tool/configs/
```

### Windows
```
C:\Users\用户名\AppData\Roaming\ios-android-refactor-tool\configs\
```

### 文件命名规则
```
平台_类型_配置名_时间戳.json

示例:
ios_variable_生产环境_1707122400000.json
android_variable_测试环境_1707036000000.json
ios_image_Google Play_1706949600000.json
```

## ❓ 常见问题

### Q1: 配置文件可以在不同电脑之间共享吗？

**A:** 可以！通过以下方式：
1. 导出配置文件
2. 复制到另一台电脑
3. 在另一台电脑导入

### Q2: 配置文件会丢失吗？

**A:** 不会。配置文件保存在系统目录，除非手动删除。
建议定期导出备份重要配置。

### Q3: 可以编辑配置文件吗？

**A:** 可以！配置文件是 JSON 格式，可以用文本编辑器编辑。
但建议通过工具界面修改，更安全。

### Q4: 模板可以自定义吗？

**A:** 当前版本使用内置模板。
可以基于模板修改后保存为自己的配置。

### Q5: 最多可以保存多少个配置？

**A:** 没有限制！可以保存任意多个配置。
